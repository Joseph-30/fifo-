# Asynchronous FIFO Physical Design (Sky130)

This repository contains the RTL, physical design flow, and verification scripts for a parameterized Asynchronous FIFO based on the Cummings architecture. The physical design (RTL-to-GDSII) is implemented using the open-source **LibreLane** framework targeting the **Skywater 130nm (Sky130)** Process Design Kit (PDK).

---

## 📌 Project Overview

An Asynchronous FIFO safely transfers data between two distinct, unsynchronized clock domains. This implementation features:
* **Cummings Architecture:** Uses Gray code pointers to safely pass read/write addresses across clock boundaries.
* **Parameterized Design:** Configurable `DATASIZE` (word width) and `ADDRSIZE` (FIFO depth).
* **Dual-Port RAM:** Built-in fall-through memory module.
* **Full Physical Layout:** Synthesized, placed, and routed using OpenROAD/Yosys via LibreLane.
* **Post-Layout Verification:** Fully verified against real-world parasitic delays using Gate-Level Simulation (GLS) with Standard Delay Format (SDF) annotation.

---

## 📂 File Structure

* `async_fifo.v` - The core RTL containing the top wrapper, synchronizers, empty/full logic, and memory.
* `tb_async_fifo.v` - Testbench for both RTL and Post-Layout Gate-Level Simulation.
* `librelane_flow.ipynb` - Google Colab notebook containing the complete LibreLane flow to generate the GDSII layout.
* `/gls/` - (Directory) Will contain the post-routing netlist and SDF delay files generated by the flow.

## 🚀 RTL-to-GDSII Flow (LibreLane)

The physical design is automated using LibreLane. You can run the provided Python script/Colab notebook to push the RTL through the following stages:

1.  **Configuration:** Target `async_fifo`, define both `wclk` and `rclk` as clock ports, and set a target clock period.
2.  **Synthesis (Yosys):** Maps the Verilog to Sky130 standard cells.
3.  **Floorplanning & IO Placement:** Defines the chip area, aligns the placement grid, and places I/O pins.
4.  **PDN Generation:** Creates the power distribution network (VDD/VSS straps).
5.  **Placement & CTS:** Places standard cells and synthesizes the clock tree to minimize clock skew across the dual domains.
6.  **Routing:** Global and detailed routing of physical metal wires.
7.  **Signoff (Magic/Netgen):** Extracts parasitics (SPEF/SDF), runs Design Rule Checks (DRC), and verifies Layout vs. Schematic (LVS).

## ✅ Verification & Simulation

### 1. Static Layout Checks
The LibreLane flow automatically performs physical signoff. If successful, the final output guarantees:
* **DRC Pass:** No spacing or manufacturing violations.
* **LVS Pass:** The routed physical layout perfectly matches the synthesized Verilog netlist (no shorts or opens).

### 2. Post-Layout Gate-Level Simulation (GLS)
To verify the timing and functionality of the final physical layout, we run a Gate-Level Simulation using the routed netlist and the extracted SDF (Standard Delay Format) file.

**Step 1: Update Testbench for SDF Annotation**
Ensure your testbench (`tb_async_fifo.v`) includes the SDF annotation block targeting your instantiated module:

```verilog
initial begin
    $dumpfile("post_layout_waves.vcd");
    $dumpvars(0, tb_async_fifo);
    
    // Inject physical delays from routing
    $sdf_annotate("path/to/librelane/output.sdf", uut);
end
```
**Step 2: Compile and Run (Icarus Verilog)**
Compile the testbench alongside the Sky130 primitives and the post-PnR netlist:


# Compile
```verilog
iverilog -o post_layout_sim.vvp \
  tb_async_fifo.v \
  path/to/librelane/post_route_netlist.v \
  /path/to/sky130A/libs.ref/sky130_fd_sc_hd/verilog/primitives.v \
  /path/to/sky130A/libs.ref/sky130_fd_sc_hd/verilog/sky130_fd_sc_hd.v
```
# Execute Simulation
```verilog
vvp post_layout_sim.vvp
```
**Step 3: View Waveforms**
Open the generated .vcd file in GTKWave to verify the read/write transactions against the physical clock delays:
```verilog
gtkwave post_layout_waves.vcd
```
